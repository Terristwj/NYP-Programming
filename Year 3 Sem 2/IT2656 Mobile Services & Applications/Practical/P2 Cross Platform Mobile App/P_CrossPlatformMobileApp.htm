Practical 2:
Cross Platform Mobile App

Objectives:

Run the app on a mobile emulator / device.

      

 

Tasks:                  

1.    Host mobile app using Firebase.

2.    Run mobile app on Android

3.    Run mobile app on iOS

 

 

1         Firebase

1.1      Create Firebase Project

1.    Go to https://firebase.google.com.

2.    Click Get Started.

3.    Sign in using your Google Account.

4.    Select Add project.

5.    Fill in the Project name and location. Click Create Project.

 

1.2      Enable Firebase Hosting

1.    Select Hosting tab.

2.    Click Get Started.
You may follow the firebase console instructions. For convenience, the steps are duplicated below also.

3.    Open Visual Studio Code. Open Terminal.

To host your site with Firebase Hosting, you need to install the Firebase CLI
Windows:

npm install -g firebase tools

        Mac:

sudo npm install -g firebase tools

 

4.    Make sure you are in your Ionic App folder.

cd D:\IonicApps\skippyQ

Sign in to Google

firebase login

Initiate your project.

firebase init

Select the SkippyQ project in Firebase.

5.    Select Hosting by using the space key.
Press Enter to confirm.

6.    Build your production-ready Ionic App.

ionic build --prod

The www folder will be built.

7.    Deploy your mobile web app.

firebase deploy

8.    Type www as the public folder.

9.    Click Continue to console.

10.  You should now see your web app on the World Wide Web by going to your domain such as https://skippy-q.web.app . Open the website in your mobile phone to verify that it looks good on a mobile web browser.

2         Run on Android

2.1   Add Android Platform

Prerequisites: Java JDK 8 and Android SDK installed. The appropriate directories must also have been added to the environment variables JAVA_HOME, ANDROID_SDK_ROOT and PATH.

1.    Go to the skippyQ folder using the  cd  command.

cd IonicApps/skippyQ

 

2.    Type the following command to add an android platform if you have not done so yet.

ionic cordova platform add android

If you are running for the first time, files will be downloaded. Please be patient. 

3.    Explore the project using Finder in Mac or File Explorer in Windows. A new platforms/android subfolder is added.

 

4.    Type the following command to copy over the web assets for a native android app.

ionic cordova prepare android

 

2.2   Run in Android

There are 3 options to run android.

Option 1: Android Studio

This is the most stable and recommended way.

1.     Import the project into Android Studio. Select File à Import Project. Select the folder skippyQ > platforms > android.

Two projects android and CordovaLib will be imported.

2.     In Android Studio, run the project by selecting Run on the toolbar.

Option 2: Command Line Interface (CLI)

1.     Start the Android Virtual Device (AVD) by clicking on the toolbar to bring up the AVD Manager.

 

2.     Type the following command to emulate the application on an Android Virtual Device (AVD).

ionic cordova emulate android

 

(Optional) If you have an Android device plugged in via USB wire and have enabled USB debugging on the device

3.     Run the application on the device using the following command.

ionic cordova run android

Option 3: Copy Apk File

(Optional) Only applicable if you have an Android device plugged in via USB wire and have enabled USB debugging on the device.

1.     If you've updated your src files, type the following command to build the android files.

ionic cordova build android

 

2.     Using File Explorer or Finder, go to the folder skippyQ > platforms > android > build > outputs > apk. Copy the file android-debug.apk to your Android device.

3.     Using Files Explorer on your Android device, click on the file android-debug.apk.

4.     When promoted, select allow unknown sources for this instance only.

 

3    Run on iOS

Skip this task if you are using a Windows computer.


Prerequisites: Xcode installed on Mac machines.

3.1   Add iOS Platform

1.    Type the following command to add an iOS platform.

ionic cordova platform add ios

If you are running for the first time, files will be downloaded. Please be patient.

2.    Explore the platforms folder. Notice that an ios sub-folder has been created.

3.    Type the following command to copy over the web assets for a native iOS app.

ionic cordova prepare ios

 

3.2   Run in iOS

There are 2 options to run iOS.

Option 1: XCode

1.     You may build and run the project in XCode by opening skippyQ.xcodeproj located in the directory platforms/ios.

2.     Run the project by clicking on the Run button in XCode

 

Option 2: Command Line Interface (CLI)

1.     Ensure that you have install the ios command line utilities.

npm install -g ios-sim

npm install -g ios-deploy

 

2.     Type the following command to emulate the application on an iOS simulator.

ionic cordova emulate ios

 

 (Optional) If you have set up the iOS provisioning profiles in XCode and an iOS device is plugged in via USB,

3.     Run the application on the device using the following command.

ionic cordova run ios
